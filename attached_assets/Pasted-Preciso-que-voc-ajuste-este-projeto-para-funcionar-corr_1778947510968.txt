Preciso que você ajuste este projeto para funcionar corretamente na Vercel usando o GitHub como fonte.

Contexto:
O deploy na Vercel está falhando porque o pnpm-lock.yaml está desatualizado em relação ao package.json.

Erro da Vercel:
ERR_PNPM_OUTDATED_LOCKFILE Cannot install with "frozen-lockfile" because pnpm-lock.yaml is not up to date with <ROOT>/artifacts/pagina-de-vendas/package.json

Failure reason:
specifiers in the lockfile don't match specifiers in package.json:
1 dependency was removed: @workspace/api-client-react@workspace:*

Objetivo:
Deixar o projeto pronto para eu mandar para o GitHub e depois fazer o deploy na Vercel sem erro.

Faça exatamente estas tarefas:

1. Verifique a estrutura do projeto
Confirme qual é a pasta principal da página.
Verifique onde estão:
package.json
pnpm-lock.yaml
index.html
src
vite.config.js ou vite.config.ts
vercel.json

2. Corrija o pnpm-lock.yaml
Rode no Shell:
pnpm install --no-frozen-lockfile

Isso deve atualizar o pnpm-lock.yaml de acordo com o package.json atual.

3. Verifique o build
Depois rode:
pnpm run build

Se der erro, corrija apenas o necessário para o build passar.

Importante:
Não altere visual da página.
Não altere copy.
Não altere imagens.
Não altere cores.
Não altere layout.
Não altere botões.
Não altere links do checkout.
Não altere lógica de UTMs.

4. Verifique o arquivo vercel.json
Se não existir, crie um vercel.json compatível com Vercel.
Se for Vite/React, use algo assim:

{
  "installCommand": "pnpm install --no-frozen-lockfile",
  "buildCommand": "pnpm run build",
  "outputDirectory": "dist",
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}

Mas antes confirme se o output correto realmente é dist.

5. Verifique o package.json
Garanta que exista o script:
"build": "vite build"

E também:
"dev": "vite"
"preview": "vite preview"

Não adicione dependências desnecessárias.

6. Verifique se o Git está apontando para o GitHub correto
Rode:
git remote -v

O repositório correto é:
https://github.com/tiagoauceno/sermonpack.git

Se estiver errado, ajuste com:
git remote set-url origin https://github.com/tiagoauceno/sermonpack.git

7. Prepare o envio para o GitHub
Depois de corrigir tudo, rode:
git status

Se houver arquivos modificados, faça:
git add .
git commit -m "prepare project for vercel deploy"
git push origin main

8. Confirme o commit novo
Depois rode:
git log --oneline -5

Preciso que você confirme que existe um commit novo depois do commit antigo d33cd62.

- se pnpm install --no-frozen-lockfile rodou com sucesso
- se pnpm run build passou
- quais arquivos foram alterados
